Intro

Connect IQ is the first-ever open platform for third-party developers to create apps for Garmin products. With Connect IQ, you can build on the features that have made Garmin products an integral part of customers’ active lifestyles:

  • Customize the base watch experience with colorful watch faces
  • Translate rich sensor data into new metrics and insights with custom data fields
  • Provide information at a glance with widgets
  • Offer solutions for active lifestyles with apps
  • Manage and playback audio content, like music, podcasts, and audio books, from third-party content providers.

Connect IQ apps are developed in Monkey C, an object-oriented language built from the ground up for easy Connect IQ app development on Garmin devices, allowing developers to focus more on the customer and less on resource constraints.

To find out more about Connect IQ and Monkey C, take a look at:

  • The Developer Site at http://developer.garmin.com/connect-iq/
  • The Programmer’s Guide, available locally or at https://developer.garmin.com/connect-iq/programmers-guide/
  • The API Docs available locally or at https://developer.garmin.com/connect-iq/api-docs/
  • The Connect IQ developer forums at https://forums.garmin.com/developer/connect-iq/

Thank you for trying Connect IQ!

Getting Started

These instructions will help you get set up and working quickly, but for more in-depth instructions, refer to the Getting Started chapter of the Connect IQ Programmer’s Guide.

Install Java

Before getting started with the installation, you’ll need Oracle Java™ Runtime Environment 8 (version 1.8.0 or higher) installed. There is a step-by-step guide on the Java website here.

Install the Eclipse Plug-in

The Connect IQ Eclipse plug-in turns Eclipse into a Connect IQ development environment. It requires the Eclipse IDE for Java Developers (either 32 or 64 bit, Eclipse Mars or newer) and a compatible Java runtime environment. Once Eclipse is installed, here’s how to install the plug-in:

  1. In Eclipse, click the Help menu
  2. Choose Install New Software…
  3. Click the Add… button
  4. Add https://developer.garmin.com/downloads/connect-iq/eclipse/ to the Location field and click OK
  5. Check the box next to Connect IQ in the Available Software window and click Next
  6. Review the license agreement and click Finish
  7. Once the installation completes, restart Eclipse

Install the SDK

Once the plug-in is installed, you’re ready to install the Connect IQ SDK:

  1. In Eclipse, click the Connect IQ menu
  2. Choose Open SDK Manager
  3. Click the Download button next to the latest SDK release
  4. Review the license agreement, and click Agree
  5. Once the download completes, click Yes when prompted to use the new SDK version as your active SDK
  6. Close the SDK Manager

Now you’re ready to import a sample project (check out the Samples directory included with the SDK) or start your own, new project. Be sure to review the Programmer’s Guide for more info about building and publishing your projects!

v0.0.13

  • Added Date/Calendar support. There are two new classes in the Toybox::Time module - Moment and Duration. A Moment represents a fixed point in time, while a Duration represents a period of time.

The Toybox::Time::Gregorian module allows the conversion from Gregorian date constructs to Moment objects. This allows for getting a Moment as follows:

// When the Flux capacitor was conceived
var date = Toybox.Time.Gregorian.moment( {:year=>1955, :month=>:november, :day=>5} );
var duration = Toybox.Time.Gregorian.duration( {:days=>5} );
var leaveDate = date.add( duration );

Gregorian was separated from the Time module to support other calendar formats, though admittedly, it would take serious effort to support anything other than Julian, and I don’t see the rise of the Roman empire coming anytime soon.

  • Resources are no longer loaded by default - a page has to explicitly load them. To load a bitmap or font, use Toybox.WatchUi.loadResource().
var resource = Toybox.WatchUi.loadResource( Rez.ID_RESOURCE );

Once the instance exists, you can reference it where you formerly passed in the resource ID.
